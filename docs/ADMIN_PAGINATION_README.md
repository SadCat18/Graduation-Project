# 后台分页实现说明（每页 10 条）

## 目标
- 后台管理各模块统一分页展示。
- 每页固定 `10` 条数据，避免列表过长影响操作效率。

## 已覆盖模块
- 用户管理（`users`）
- 帖子管理（`posts`）
- 评论管理（`comments`）
- 活动管理（`activities`）
- 内容管理下的公告、资讯、场地、轮播

## 实现位置
- 前端页面：`frontend/src/views/AdminPage.vue`

## 实现方式
### 1) 服务端分页模块（用户/帖子/评论/活动）
- 分页参数统一为：`page` + `size=10`
- 接口返回结构使用已有 `PageResult`：`{ total, list }`
- 页面状态保存：
  - 当前页 `pageState.xxx.page`
  - 总条数 `pageState.xxx.total`
- 页码通过「上一页 / 下一页」按钮切换。

### 2) 内容管理模块（公告/资讯/场地/轮播）
- 当前接口是一次性返回全量列表（未分页接口）。
- 前端采用本地分页切片展示：
  - `sliceByPage(list, page)` 按 10 条切片
  - `contentPageState` 保存各子模块页码
- 同样提供「上一页 / 下一页」分页控件。

## 关键常量
- `ADMIN_PAGE_SIZE = 10`

## 注意事项
- 删除最后一条数据后，会自动回退到上一页（防止空白页）。
- 若后续要改成每页 20 条，只需修改 `ADMIN_PAGE_SIZE`。
